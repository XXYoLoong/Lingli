# 多端适配参考指南

本指南旨在为鸿蒙应用开发提供多端适配的最佳实践参考。基于本项目的实际代码实现，总结了一套成熟的响应式布局与资源适配方案。

## 1. 核心思想

多端适配的核心在于**“一次开发，多端部署”**。通过响应式布局技术，使应用能够自动适应手机、折叠屏、平板、2in1 设备等不同屏幕尺寸，提供一致且优化的用户体验。

主要采用以下三种策略：
1.  **断点系统 (Breakpoint System)**：监听屏幕宽度，感知设备形态变化。
2.  **栅格布局 (Grid Layout)**：基于断点自动调整页面结构。
3.  **资源适配 (Resource Adaptation)**：**【推荐】** 利用系统能力，根据设备类型自动加载对应资源。
4.  **动态属性配置 (Dynamic Configuration)**：**【备选】** 在代码中根据断点手动调整参数。

## 2. 断点系统 (Breakpoint System)

本项目封装了一套基于 `mediaquery` 的断点监听系统，定义了标准的屏幕宽度区间。

### 2.1 断点定义

| 断点名称 | 屏幕宽度范围 (vp) | 典型设备 |
| :--- | :--- | :--- |
| **xs** | [0, 320) | 超小屏设备 |
| **sm** | [320, 600) | 普通手机 |
| **md** | [600, 840) | 折叠屏展开态、小平板 |
| **lg** | [840, 1440) | 大平板、2in1 设备 |
| **xl** | [1440, +∞) | 大屏显示器 |
| **xxl** | (预留) | 超大屏 |

### 2.2 实现代码

你可以直接复制以下代码到项目工具类文件（如 `BreakpointSystem.ets`）中使用：

```typescript
// common/breakpointsystem.ets
import { mediaquery } from '@kit.ArkUI';


export type BreakpointType = 'xs' | 'sm' | 'md' | 'lg' | 'xl' | 'xxl';


export interface Breakpoint {
  name: BreakpointType;
  size: number;
  mediaQueryListener?: mediaquery.MediaQueryListener;
}


export class BreakpointSystem {
  private static instance: BreakpointSystem;
  private readonly breakpoints: Breakpoint[] = [
    { name: 'xs', size: 0 },
    { name: 'sm', size: 320 },
    { name: 'md', size: 600 },
    { name: 'lg', size: 840 },
    { name: 'xl', size: 1440 }
  ]
  private states: Set<BreakpointState<Object>>;


  private constructor() {
    this.states = new Set();
  }


  public static getInstance(): BreakpointSystem {
    if (!BreakpointSystem.instance) {
      BreakpointSystem.instance = new BreakpointSystem();
    }
    return BreakpointSystem.instance;
  }


  public attach(state: BreakpointState<Object>): void {
    this.states.add(state);
  }


  public detach(state: BreakpointState<Object>): void {
    this.states.delete(state);
  }


  public start() {
    this.breakpoints.forEach((breakpoint: Breakpoint, index) => {
      let condition: string;
      if (index === this.breakpoints.length - 1) {
        condition = `(${breakpoint.size}vp<=width)`;
      } else {
        condition = `(${breakpoint.size}vp<=width<${this.breakpoints[index + 1].size}vp)`;
      }
      breakpoint.mediaQueryListener = mediaquery.matchMediaSync(condition);
      if (breakpoint.mediaQueryListener.matches) {
        this.updateAllState(breakpoint.name);
      }
      breakpoint.mediaQueryListener.on('change', (mediaQueryResult) => {
        if (mediaQueryResult.matches) {
          this.updateAllState(breakpoint.name);
        }
      })
    })
  }


  private updateAllState(type: BreakpointType): void {
    this.states.forEach(state => state.update(type));
  }


  public stop() {
    this.breakpoints.forEach((breakpoint: Breakpoint, index) => {
      if (breakpoint.mediaQueryListener) {
        breakpoint.mediaQueryListener.off('change');
      }
    })
    this.states.clear();
  }
}


export interface BreakpointOptions<T> {
  xs?: T;
  sm?: T;
  md?: T;
  lg?: T;
  xl?: T;
  xxl?: T;
}


export class BreakpointState<T extends Object> {
  public value: T | undefined = undefined;
  private options: BreakpointOptions<T>;


  constructor(options: BreakpointOptions<T>) {
    this.options = options;
  }


  static of<T extends Object>(options: BreakpointOptions<T>): BreakpointState<T> {
    return new BreakpointState(options);
  }


  public update(type: BreakpointType): void {
    if (type === 'xs') {
      this.value = this.options.xs;
    } else if (type === 'sm') {
      this.value = this.options.sm;
    } else if (type === 'md') {
      this.value = this.options.md;
    } else if (type === 'lg') {
      this.value = this.options.lg;
    } else if (type === 'xl') {
      this.value = this.options.xl;
    } else if (type === 'xxl') {
      this.value = this.options.xxl;
    } else {
      this.value = undefined;
    }
  }
}
```

## 3. 栅格布局 (Grid Layout)

`GridRow` 和 `GridCol` 是实现响应式布局的核心组件。

### 3.1 基础配置

在根容器使用 `GridRow`，并显式配置断点和列数：

```typescript
GridRow({
  columns: {
    sm: 4,  // 手机：4列
    md: 8,  // 折叠屏/平板：8列
    lg: 12, // 大屏：12列
    xl: 12,
    xxl: 12
  },
  breakpoints: {
    value: ['320vp', '600vp', '840vp', '1440vp'],
    reference: BreakpointsReference.WindowSize
  }
}) {
  // GridCol 内容...
}
.onBreakpointChange((breakpoints: string) => {
  this.currentBreakpoint = breakpoints;
})
```

### 3.2 响应式跨度 (Span)

通过 `GridCol` 的 `span` 属性控制组件在不同屏幕下的宽度：

```typescript
// 主内容区域
GridCol({
  span: {
    sm: 4, // 手机全宽
    md: 4, // 平板半宽
    lg: 2, // 大屏 2/12
    xl: 2
  }
}) {
  this._Main();
}

// 侧边栏（仅中大屏显示）
GridCol({
  span: {
    sm: 0, // 手机隐藏
    md: 0, 
    lg: 2, // 大屏显示
    xl: 2
  }
}) {
  if (this.currentBreakpoint != "sm" && this.currentBreakpoint != "md") {
    this._SideBar();
  }
}
```

## 4. 资源适配 (Resource Adaptation) —— **【首选推荐】**

**原则：能用资源适配解决的，坚决不要写代码判断断点。**

利用鸿蒙系统的资源限定词目录结构，针对不同设备加载特定的资源文件（图片、布局参数等）。这种方式将适配逻辑下沉到系统层面，代码更加简洁、易维护，且性能更好。

### 4.1 目录结构

在 `src/main/resources` 下建立限定词目录：

*   `resources/base`: 默认资源（兜底）
*   `resources/phone`: 手机专用（也可以用 `phone-mdpi` 等）
*   `resources/tablet`: 平板专用
*   `resources/2in1`: 2in1 设备专用

### 4.2 使用示例

假设我们需要定义一个标题的字体大小，在不同设备上不一样：

**Step 1: 定义资源文件**

在 `resources/base/element/float.json` (默认):
```json
{
  "float": [
    { "name": "main_title_font_size", "value": "18fp" }
  ]
}
```

在 `resources/tablet/element/float.json` (平板):
```json
{
  "float": [
    { "name": "main_title_font_size", "value": "24fp" }
  ]
}
```

**Step 2: 代码调用**

```typescript
Text("我的标题")
  .fontSize($r('app.float.main_title_font_size')) // 系统会自动根据设备类型加载对应值
```

可以看到，代码中完全不需要 `if (isTablet)` 这样的判断逻辑。

## 5. 动态属性配置 (Dynamic Configuration) —— **【次选方案】**

**注意：** 只有当 **方法 4 (资源适配)** 无法满足需求时（例如需要极其复杂的动态计算逻辑），才考虑使用本方法。尽量减少在代码中维护大量的尺寸映射表。

### 5.1 配置模式

创建一个对象，Key 为组件标识，Value 为包含各断点数值的对象：

```typescript
// Size.ts
export const ComponentSize = {
  // 这种写法虽然灵活，但不如资源文件直观，且难以维护
  bannerHeight: {
    sm: 106,
    md: 128,
    lg: 150,
    xl: 250,
    xxl: 300
  },
  titleFontSize: {
    sm: 18,
    md: 20,
    lg: 24,
    xl: 30
  }
}
```

### 5.2 UI 使用

在组件中根据当前断点读取值：

```typescript
import { ComponentSize } from '../data/Size';

// ... inside component
Image($r('app.media.banner'))
  .height(ComponentSize.bannerHeight[this.currentBreakpoint] || 106)
```

**缺点：**
1.  代码耦合度高。
2.  需要手动监听断点变化并刷新 UI。
3.  不如系统资源加载机制高效。

## 6. 导航栏响应式布局 (Responsive Navigation Layout)

本项目在主页 (`Index.ets`) 实现了导航栏位置的自适应调整：在移动端/平板端显示为**底部导航**，在超大屏 (xl) 设备上自动切换为**侧边导航**。

### 6.1 实现思路

利用 `Tabs` 组件的 `vertical` 和 `barPosition` 属性，结合断点系统动态控制导航栏的形态。

*   **小屏/中屏 (sm, md, lg)**: 底部导航
    *   `vertical`: `false` (水平排列)
    *   `barPosition`: `BarPosition.End` (底部)
*   **超大屏 (xl)**: 侧边导航
    *   `vertical`: `true` (垂直排列)
    *   `barPosition`: `BarPosition.Start` (起始位置/左侧)

### 6.2 代码示例

1.  **定义状态映射**：
    使用 `BreakpointState` 将断点映射为布尔值。`true` 代表侧边导航模式 (xl)，`false` 代表底部导航模式。

    ```typescript
    // Index.ets
    @State compSize: BreakpointState<boolean> = BreakpointState.of({
      sm: false,
      md: false,
      lg: false,
      xl: true // 仅在 xl (>=1440vp) 启用侧边导航
    });

    aboutToAppear(): void {
      // 绑定断点监听
      BreakpointSystem.getInstance().attach(this.compSize);
      BreakpointSystem.getInstance().start();
    }
    ```

2.  **绑定 Tabs 属性**：
    在 `build` 方法中，将 `compSize.value` 绑定到 `Tabs` 的关键属性上。

    ```typescript
    build() {
      // barPosition: 0 (Start/Left) or 1 (End/Bottom)
      Tabs({
        barPosition: this.compSize.value ? BarPosition.Start : BarPosition.End,
        index: this.currentIndex
      }) {
        // TabContent...
      }
      .vertical(this.compSize.value) // true: 垂直模式 (侧边栏), false: 水平模式 (底部栏)
      // ...
    }
    ```

## 7. 总结

| 场景 | 推荐方案 | 说明 |
| :--- | :--- | :--- |
| **数值/字符串/颜色** | **资源适配 (Method 4)** | 使用 `float.json`, `string.json`, `color.json` |
| **图片/图标** | **资源适配 (Method 4)** | 使用 `media` 目录下的限定词目录 |
| **整体布局结构** | **栅格布局 (GridRow)** | 配合断点系统调整列数和 Span |
| **导航栏形态** | **组件属性动态绑定** | 利用 `Tabs.vertical` 配合断点切换底部/侧边栏 |
| **复杂逻辑/动态计算** | **动态属性 (Method 5)** | 仅作为最后手段使用 |

在开发新功能时，请优先思考：**“这个差异能不能通过配置 `resources` 目录下的资源文件来解决？”** 如果能，就不要写 TypeScript 代码来判断。
